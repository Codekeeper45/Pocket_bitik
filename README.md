# 🤖 Pocket Bitik — умный Telegram-помощник на базе ИИ

**Pocket Bitik** — это «userbot» (юзербот) для Telegram. Он работает **от имени вашего личного аккаунта** и помогает прямо в переписке:

- 💬 **`/ask`** — задайте вопрос ИИ по последним сообщениям чата («о чём тут спорили?», «сделай выжимку», «кто что обещал?»). Длинные ответы приходят **свёрнутой цитатой** — не засоряют чат, раскрываются тапом.
- 🖼️ Понимает **картинки, голосовые и видео-кружки** в переписке (распознаёт и описывает их).
- 🎨 **`/gen`** — **генерирует и редактирует изображения** (бесплатная модель; промпт может составить ИИ по контексту чата).
- 📡 Следит за **Telegram-каналами**, делает **поиск** по ним и присылает **дайджест**.
- 🔁 **Авто-ответ** на сообщения с памятью диалога.
- 🧠 Поддерживает много моделей ИИ: **DeepSeek, Kimi, GLM, Qwen, MiniMax, Gemini** и др.

> Управление — короткими командами, которые вы пишете сами себе или в чате, например `/ask 100 сделай выжимку`. Команды видите только вы.

---

## ⚠️ Важно прочитать перед началом

1. **Это юзербот.** Он входит в Telegram под **вашим аккаунтом** (не отдельный бот с `@username`). Все команды вы пишете сами от своего имени.
2. **Один запуск на один аккаунт.** Нельзя запускать бота в двух местах одновременно с одной и той же сессией — Telegram разлогинит сессию. Запускайте только в одном месте.
3. **Берегите файлы `.env` и `*.session`.** Файл сессии = полный доступ к вашему Telegram. **Никому его не отправляйте и не выкладывайте в интернет.** Эти файлы специально исключены из репозитория (см. `.gitignore`).
4. **Юзерботы формально против правил Telegram.** Используйте на свой страх и риск, на своём личном аккаунте. Не используйте для спама или нарушения чужих прав.
5. **ИИ стоит денег.** Запросы к ИИ платные, но очень недорогие (доли цента за запрос). Для старта достаточно пополнить пару долларов на DeepSeek.

---

## 🧰 Что понадобится

- Компьютер с **Windows, macOS или Linux** (бот может работать и на дешёвом сервере 24/7, но для начала хватит обычного компьютера).
- **Аккаунт Telegram** и телефон, на который приходят коды входа.
- **15–20 минут** времени.
- Аккаунт на **[DeepSeek](https://platform.deepseek.com)** и его API-ключ (это единственный обязательный ИИ-ключ). OpenRouter и OpenCode — по желанию, для дополнительных функций.

Ничего программировать не нужно — просто по шагам выполняйте команды ниже.

## 🚀 Установка по шагам

> Команды ниже нужно вводить в **терминале** (командной строке).
> - **Windows:** нажмите «Пуск» → наберите `PowerShell` → откройте «Windows PowerShell».
> - **macOS:** «Программы» → «Утилиты» → «Терминал».
> - **Linux:** ваш привычный терминал.

### Шаг 1. Установите Python (версия 3.10 или новее)

Python — это «движок», на котором работает бот.

- **Windows:**
  1. Откройте [python.org/downloads](https://www.python.org/downloads/) и нажмите большую жёлтую кнопку **«Download Python 3.x.x»** вверху.
  2. Запустите скачанный файл. В самом первом окне установщика **обязательно поставьте галочку внизу «Add python.exe to PATH»** (без неё ничего не заработает), затем нажмите **«Install Now»** и дождитесь конца.
  3. **Важно:** после установки **закройте окно терминала (PowerShell) и откройте заново** — иначе система ещё «не видит» Python.
- **macOS:** откройте [python.org/downloads](https://www.python.org/downloads/), нажмите жёлтую кнопку «Download Python 3.x.x», запустите скачанный `.pkg` и пройдите установку «Далее → Далее». (Или, если есть Homebrew: `brew install python`.)
- **Linux (Ubuntu/Debian):** в терминале выполните `sudo apt update && sudo apt install python3 python3-venv python3-pip -y`

Проверьте, что установилось. Введите в терминале команду и нажмите Enter:

```bash
python --version
```
В ответ должно появиться что-то вроде `Python 3.12.1` (главное — чтобы число было **3.10 или больше**).
Если пишет «команда не найдена» / «is not recognized» — попробуйте `python3 --version`. Если и так не работает на Windows — вы пропустили галочку «Add to PATH» (шаг 1.2): переустановите Python и поставьте её.

> 💡 Дальше в инструкции везде пишется `python`. Если у вас сработал только `python3` — значит, и дальше везде вместо `python` пишите `python3` (и вместо `pip` — `pip3`).

### Шаг 2. Скачайте код бота

**Простой способ (без программ):**
1. Откройте в браузере страницу репозитория: **https://github.com/Codekeeper45/Pocket_bitik**
2. Найдите зелёную кнопку **`< > Code`** (справа, над списком файлов), нажмите её → в выпавшем меню выберите **`Download ZIP`**.
3. Дождитесь скачивания, найдите ZIP-архив в папке «Загрузки» и **распакуйте** его (правый клик → «Извлечь всё» на Windows / двойной клик на macOS) в удобное место, например на рабочий стол.
   > После распаковки папка будет называться `Pocket_bitik-main` (с приставкой `-main`) — это нормально. Можете переименовать её в `Pocket_bitik`. Откройте папку — внутри должен лежать файл **`bot_new.py`**. Если внутри оказалась ещё одна папка с тем же именем — заходите в неё, работать нужно с той, где лежит `bot_new.py`.

**Способ для тех, у кого есть Git:**
```bash
git clone https://github.com/Codekeeper45/Pocket_bitik.git
cd Pocket_bitik
```

### Шаг 3. Откройте терминал внутри папки с ботом

- **Windows:** откройте папку с ботом в Проводнике. Кликните один раз по **адресной строке** вверху (длинная белая строка, где показан путь к папке, например `C:\Users\...\Pocket_bitik`) — текст выделится. Сотрите его, напишите `powershell` и нажмите Enter. Откроется чёрное/синее окно — это и есть терминал, уже в нужной папке.
- **macOS:** правый клик по папке → «Новые службы» → «Новый терминал в папке». Если такого пункта нет — откройте «Терминал», напечатайте `cd ` (со space в конце) и **перетащите папку с ботом в окно терминала**, затем нажмите Enter.
- **Linux:** правый клик в папке → «Открыть в терминале».

Чтобы убедиться, что вы в нужной папке, введите `ls` (macOS/Linux) или `dir` (Windows) и нажмите Enter — в списке вы должны увидеть файл `bot_new.py`. Если его нет — вы открыли терминал не в той папке (вернитесь к началу этого шага).

> ⚠️ **Важно:** все команды из следующих шагов вводятся **в этом же окне терминала**. Не закрывайте его до конца установки.

### Шаг 4. Создайте «виртуальное окружение» и установите зависимости

Это изолированное место для библиотек бота, чтобы ничего не сломать в системе.

**Windows (PowerShell):**
```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```
> Если PowerShell ругается на запуск скриптов, один раз выполните:
> `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` (нажмите `Y`), потом повторите активацию.

**macOS / Linux:**
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Вводите эти три строки **по одной**, нажимая Enter после каждой и дожидаясь, пока команда закончится.

- После второй строки (активации) в начале строки терминала появится приписка **`(.venv)`** — это значит, что окружение включилось. Так и должно быть.
- Последняя строка (`pip install …`) скачивает библиотеки: это занимает **1–3 минуты**, по экрану побежит много текста — **это нормально**, не закрывайте окно. Дождитесь, пока снова появится строка для ввода (с мигающим курсором).

### Шаг 5. (По желанию) Установите ffmpeg

`ffmpeg` нужен **только** чтобы бот понимал **видео-кружки** (те самые круглые видео). Голосовые и картинки работают без него. **Если кружки вам не нужны — спокойно пропустите этот шаг**, бот без `ffmpeg` запустится нормально.

- **Windows (проще всего):** в терминале выполните `winget install ffmpeg`, дождитесь конца, затем **закройте и снова откройте терминал** (как в Шаге 1) и заново активируйте окружение (Шаг 4). Если `winget` не найден — этот шаг лучше пропустить (ручная установка с [ffmpeg.org](https://ffmpeg.org/download.html) сложна и здесь не описывается).
- **macOS:** `brew install ffmpeg` (нужен установленный Homebrew).
- **Linux:** `sudo apt install ffmpeg -y`

### Шаг 6. Получите ключи Telegram (`api_id` и `api_hash`)

1. Откройте в браузере **[my.telegram.org](https://my.telegram.org)**.
2. Введите свой номер телефона (в международном формате, например `+79991234567`) и нажмите «Next». В приложение Telegram придёт сообщение с **кодом** от «Telegram» — введите его на сайте.
3. На открывшейся странице нажмите **«API development tools»**.
4. Заполните форму **только латиницей** (без кириллицы и эмодзи):
   - **App title:** короткое название латиницей, например `Pocket Bitik`. Не используйте слово «telegram» и спецсимволы (`@ # ...`).
   - **Short name:** **только латинские буквы и цифры**, длина **5–32**, **без пробелов, дефисов и подчёркиваний**. Например `pocketbitik`. ⚠️ Именно нарушение этого правила (пробел, кириллица, слишком короткое) чаще всего и вызывает «**incorrect app name**».
   - **Platform** — выберите **Desktop** (или Android). Поля **URL и Description оставьте пустыми**.
5. Нажмите кнопку **«Create application»** внизу.
6. Откроется страница, где будут два значения (возможно, придётся чуть прокрутить вниз):
   - **`App api_id`** — это **число** (например, `12345678`).
   - **`App api_hash`** — это **длинная строка** из букв и цифр (например, `0123456789abcdef0123456789abcdef`).

   Скопируйте оба в заметку — они понадобятся в Шаге 8. **Никому их не показывайте.**

> ⚠️ **Выскакивает «ERROR» или «incorrect app name»?** Сайт `my.telegram.org` печально известен капризами. Пройдите по порядку:
>
> 1. **Проверьте Short name** — только латинские буквы и цифры, 5–32 символа, **без пробелов и спецсимволов** (это причина №1 для «incorrect app name»). **App title** — без слова «telegram», без эмодзи и кириллицы.
> 2. **Выключите VPN/прокси и блокировщики рекламы (adblock).** Важно: страна вашего IP должна **совпадать со страной номера телефона** аккаунта — при несовпадении Telegram блокирует создание приложения.
> 3. Откройте сайт в **режиме инкогнито** или другом браузере (**Chrome / Firefox / Edge на компьютере**, не на телефоне), очистите кэш и куки.
> 4. **Проверьте, не создалось ли приложение уже!** Очень часто после «ERROR» оно всё равно создаётся — обновите страницу или зайдите на **[my.telegram.org/apps](https://my.telegram.org/apps)**: `api_id` и `api_hash` уже могут быть там. Тогда ошибку можно игнорировать.
> 5. **Если «ERROR» выскакивает на ЛЮБОЙ ввод (даже без VPN) — скорее всего сработал лимит после нескольких попыток.** Telegram временно блокирует повторные попытки. Закройте сайт и **подождите 30–60 минут** (можно до суток), затем попробуйте снова — с первой попытки и с правильным Short name. Не «долбите» кнопку: каждая новая попытка продлевает блокировку.
> 6. **Аккаунт должен быть «нормальным»:** не только что зарегистрированным, без ограничений/спам-флагов, желательно с заданным **@username** (Настройки → Изменить профиль → имя пользователя). Совсем новым/ограниченным аккаунтам Telegram часто не даёт создавать приложения.
> 7. Помогает также: сменить язык сайта на **English** (ссылка внизу страницы), зайти с **другой сети** (мобильный интернет вместо Wi-Fi) или с другого устройства.
> 8. **Если перепробовали всё:**
>    - Заполните **Description** короткой фразой по-английски (напр. `personal userbot`) — иногда пустое поле и есть причина «ERROR».
>    - Временно отключите в Telegram **двухфакторную защиту** (облачный пароль): Настройки → Конфиденциальность → Облачный пароль. У части людей форма проходит только так (потом пароль можно вернуть).
>    - На компьютере сделайте **сброс DNS**: Windows — `ipconfig /flushdns`, macOS — `sudo dscacheutil -flushcache`.
>    - Проверьте, не лежит ли сам Telegram ([downdetector.com/status/telegram](https://downdetector.com/status/telegram/)).
>    - В крайнем случае напишите в поддержку: **[telegram.org/support](https://telegram.org/support)** или боту **@BotSupport** (опишите проблему и что уже пробовали).
>
> 💬 **Сообщение «Подтвердите действие на my.telegram.org» в Telegram** — это штатное подтверждение входа: откройте чат с сервисным аккаунтом **«Telegram»** (синяя галочка), введите присланный оттуда **код** на сайте, а если есть кнопка подтверждения — нажмите её. Без этого форма создания приложения не откроется.

### Шаг 7. Получите ключ к ИИ (обязателен только DeepSeek)

Боту нужен **минимум один** ИИ-ключ — **DeepSeek**. Этого достаточно, чтобы бот запустился и работали основные функции. Остальные ключи — по желанию (см. таблицу ниже).

1. Зарегистрируйтесь на **[platform.deepseek.com](https://platform.deepseek.com)** (по email или через Google).
2. Пополните небольшой баланс: в левом меню откройте раздел **«Top up»** (или «Billing») и внесите пару долларов — этого надолго хватит, запросы стоят доли цента.
3. В левом меню откройте **«API keys»** → нажмите **«Create new API key»** → дайте любое имя → создать.
4. Ключ (начинается на **`sk-...`**) покажется **только один раз** — сразу скопируйте его в заметку. Если потеряли — ничего страшного, просто создайте новый.

#### Какой ключ за что отвечает

| Ключ | Обязателен? | Что включает |
|---|:---:|---|
| **DeepSeek** | ✅ Да | «Мозг» бота: `/ask`, `/search`, `/digest`, авто-ответ |
| **OpenRouter** | ⬜ Нет | Распознавание **картинок, голосовых и видео-кружков** в `/ask` + **генерация изображений** `/gen` |
| **OpenCode** | ⬜ Нет | Возможность переключаться на модели **Kimi / GLM / Qwen / MiniMax** через `/model` |
| **Google GenAI** | ⬜ Нет | **Голосовые ответы** в `/ask` (команда `/voice` + флаг `-v`, озвучка Gemini TTS) |
| **Fish Audio** | ⬜ Нет | Альтернативный TTS-движок с большим выбором голосов (`/voice engine fish`, `/voice fish`) |
| **LlamaCloud** | ⬜ Нет | Дешёвый **OCR фото** в `/ask` по умолчанию (текст с картинок); без него фото описывает vision-модель |

> 🟢 **Важно:** ключи независимы. Можно вписать **только DeepSeek** — бот **не упадёт** и будет нормально работать. Просто функции, привязанные к недостающему ключу, не сработают: например, без OpenRouter картинки в чате не распознаются (текст разбирается как обычно), а отправить голосовое на расшифровку не получится. Добавите ключ позже — функция включится сама.
>
> Ключи OpenRouter ([openrouter.ai/keys](https://openrouter.ai/keys)) и OpenCode добавляются по тому же принципу — вписываются в `.env` (Шаг 8). Если они вам не нужны — пропустите.

### Шаг 8. Создайте файл настроек `.env`

`.env` — это файл с вашими ключами. В папке уже есть готовый шаблон `.env.example`; нужно сделать его копию с именем `.env` и вписать туда свои значения.

**Шаг 8.1. Создайте копию шаблона.** Введите в терминале (в том же окне, где `(.venv)`):

**Windows (PowerShell):**
```powershell
copy .env.example .env
```
**macOS / Linux:**
```bash
cp .env.example .env
```

**Шаг 8.2. Откройте файл `.env` для редактирования.** Проще всего открыть его прямо из терминала (файл `.env` скрытый, в проводнике его можно не найти):

- **Windows:** `notepad .env`  ← откроется Блокнот
- **macOS:** `open -e .env`  ← откроется TextEdit
- **Linux:** `nano .env`  ← редактор прямо в терминале (сохранить: `Ctrl+O`, Enter; выйти: `Ctrl+X`)

**Шаг 8.3. Впишите свои значения.** Замените текст-заглушки на свои данные из Шагов 6 и 7. Минимально нужно заполнить эти строки:

```ini
api_id=сюда_число_из_шага_6
api_hash=сюда_строку_из_шага_6
DEEPSEEK_API_KEY=сюда_ключ_sk_из_шага_7
DEEPSEEK_MODEL=deepseek-v4-pro
```

Например, должно получиться так (значения — ВЫДУМАННЫЕ, подставьте свои):
```ini
api_id=12345678
api_hash=0123456789abcdef0123456789abcdef
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
DEEPSEEK_MODEL=deepseek-v4-pro
```

**🔴 Три главных правила, чтобы не сломать файл:**
1. Значение пишется **сразу после знака `=`, без пробелов**: `api_id=12345678` — правильно; `api_id = 12345678` — **неправильно**.
2. **Не ставьте кавычки** вокруг значений (никаких `"..."`).
3. Замените **весь** текст-заглушку, не оставляйте слова вроде `сюда_...` или `ВАШ_КЛЮЧ`.

Строку `DEEPSEEK_MODEL=deepseek-v4-pro` оставьте как есть. Поля `OPENROUTER_API_KEY` и `OPENCODE_API_KEY` оставьте пустыми (после `=` ничего) — бот всё равно запустится (см. таблицу в Шаге 7).

**Шаг 8.4. Сохраните файл.** Блокнот/TextEdit: «Файл → Сохранить» (или `Ctrl+S` / `Cmd+S`). В `nano`: `Ctrl+O`, Enter, затем `Ctrl+X`.

> ⚠️ **Внимание, Windows + Блокнот:** если вы создавали файл не командой `copy`, а вручную, Блокнот может незаметно сохранить его как `.env.txt`, и бот его не увидит. Поэтому используйте команду `copy .env.example .env` из шага 8.1 — она создаёт файл с правильным именем. Проверить имя можно командой `dir` — в списке должен быть файл ровно `.env` (без `.txt` на конце).

### Шаг 9. Первый вход в Telegram

Этот шаг создаёт файл сессии, чтобы бот мог входить под вашим аккаунтом без пароля каждый раз. Выполните **один раз**, в том же окне терминала (где в начале строки видно `(.venv)`):

```bash
python login.py
```

Программа задаст вопросы прямо в терминале — отвечайте, нажимая Enter после каждого:
1. **Phone number** — введите свой номер **в международном формате с плюсом**, например `+79991234567`, и нажмите Enter.
2. **Code** — в приложение Telegram придёт сообщение с кодом от «Telegram». Введите этот код в терминал и нажмите Enter.
   > Код вводится «вслепую» — символы могут не отображаться, это нормально, просто печатайте и жмите Enter.
3. Если у вас включена **двухэтапная аутентификация (облачный пароль)** — программа попросит **Password**. Введите ваш облачный пароль Telegram.

Если увидели строку **`✅ Вход выполнен`** — всё готово: рядом с `bot_new.py` появился файл `session_name.session`.

> 🔑 **Этот файл — как ключ от вашего аккаунта Telegram. Никому его не передавайте и не выкладывайте в интернет.**

### Шаг 10. Запустите бота

В том же окне терминала (с `(.venv)`) введите:

```bash
python bot_new.py
```

Через несколько секунд в терминале появятся строки, и среди них — **`Userbot запущен.`** Это значит, что бот работает! 🎉 Терминал дальше будет показывать строки-логи о работе бота — **так и должно быть, окно не закрывайте**.

Теперь откройте Telegram (на телефоне или на компьютере), зайдите в **«Избранное»** (это чат с самим собой; в английском Telegram — «Saved Messages») и напишите туда сообщение:

```
/help
```

Через миг ваше сообщение **само превратится** в список команд бота. Если так и произошло — бот вас слышит, всё настроено. 🎉

> ⛔ **Чтобы остановить бота** — вернитесь в терминал и нажмите `Ctrl + C`.
> Пока окно терминала открыто и бот запущен — он работает. Закроете окно — бот выключится.

### Шаг 11. Как запускать бота в следующий раз

Шаги 1–9 делаются **только один раз**. Когда вы потом снова захотите включить бота (например, после перезагрузки компьютера), нужно сделать всего два действия:

1. Откройте терминал **в папке с ботом** (как в Шаге 3).
2. Активируйте окружение и запустите:

**Windows (PowerShell):**
```powershell
.venv\Scripts\Activate.ps1
python bot_new.py
```
**macOS / Linux:**
```bash
source .venv/bin/activate
python3 bot_new.py
```

> 💡 Если пропустить активацию окружения (`Activate`/`source`), вы увидите ошибку вида «No module named telethon». Это значит, что окружение не активировано — выполните строку активации и запустите снова.

## 💬 Как пользоваться (команды)

Команды пишутся **вами** в любом чате (часто удобно в «Избранном»). Вот основные:

### Вопрос к ИИ по чату — `/ask`
| Команда | Что делает |
|---|---|
| `/ask 100 сделай выжимку` | Берёт последние **100** сообщений чата и отвечает на ваш вопрос |
| `/ask 500 о чём спорили?` | То же, по 500 сообщениям |
| `/ask 100 -t краткое содержание` | Флаг `-t` — **без медиа** (быстрее и дешевле, только текст) |
| `/ask 50 @ivan @anna о чём они?` | Только сообщения **от этих** людей |
| `/ask 100 !@ivan что обсуждали?` | **Исключить** сообщения Ивана |
| `/ask 30 -v расскажи анекдот` | Флаг `-v` — ответить **голосом** (озвучка, нужен ключ Google) |
| `/ask 30 -g что на фото?` | Флаг `-g` — отдать **картинки напрямую** отвечающей модели (нужна vision-модель) |
| Ответить (reply) на сообщение + `/ask вопрос` | Вопрос про **конкретное** сообщение + контекст вокруг |

Флаги можно комбинировать: `-t` (без медиа), `-c` (обязательный поиск), `-d` (сохранить контекст в файл для отладки), `-v` (ответить голосом), `-g` (картинки напрямую vision-модели), `-m` (фото описывает vision-модель; без него — дешёвый OCR, только текст с картинок). Голос всегда через Parakeet STT.

### Выбор модели ИИ — `/model`
| Команда | Что делает |
|---|---|
| `/model` | Показать список доступных моделей |
| `/model 3` | Выбрать модель номер 3 из списка |
| `/model media` | Сменить модель для распознавания картинок |
| `/cache info` | Информация о кэше распознанных медиа |
| `/cache clear all` | Очистить кэш медиа |

### Голос ответов — `/voice` (нужен ключ Google GenAI)
| Команда | Что делает |
|---|---|
| `/voice` | Список из 30 голосов; `▶` — активный |
| `/voice 16` / `/voice Kore` | Выбрать голос по номеру или имени |
| `/voice test` | Прислать пример текущим голосом |
| `/voice auto on` / `/voice auto off` | Разрешить/запретить модели самой отвечать голосом |
| `/ask 30 -v вопрос` | Разовый голосовой ответ (флаг `-v`) |
| `/voice engine fish` / `/voice engine gemini` | Сменить TTS-движок (Fish Audio / Gemini) |
| `/voice fish search <запрос>` | Найти голоса Fish Audio (нужен `FISH_AUDIO_API_KEY`) |
| `/voice fish add <id> [имя]` / `/voice fish N` | Добавить голос Fish в избранное / выбрать |
| `/model fav` / `/model remove <N>` | Избранные OpenRouter-модели / удалить кастомную |

### Генерация изображений — `/gen` (нужен ключ OpenRouter)
| Команда | Что делает |
|---|---|
| `/gen аниме кот в очках` | Сгенерировать картинку ровно по вашему промпту |
| `/gen -i закат над морем` | Флаг `-i` — ИИ сначала **улучшит** промпт |
| `/gen 100 нарисуй о чём мы спорим` | ИИ составит промпт по последним **100** сообщениям чата |
| Reply на **текстовое** сообщение + `/gen ...` | Текст сообщения уйдёт в контекст промпта (его строит ИИ) |
| Reply на **фото** + `/gen сделай фон ночным` | **Редактирование** картинки; промпт идёт дословно, `-i` подключит ИИ с учётом подписи |
| **Фото, прикреплённые** к сообщению с `/gen` (можно альбомом) | Тоже идут на вход — объединяются с фото из reply (до 10, суммарно до 3 МБ) |

Модель — Riverflow V2.5 Pro (бесплатная). Генерация может занять до пары минут.
Промпт от ИИ приходит в подписи **свёрнутой цитатой**; при временном сбое — авто-повтор,
а отклонённый провайдером ИИ-промпт ИИ сам поправит и попробует снова.

### Каналы и дайджест — `/channels`, `/search`, `/digest`
| Команда | Что делает |
|---|---|
| `/channels` | Список отслеживаемых каналов |
| `/channels add @name` | Добавить канал |
| `/channels remove @name` | Убрать канал |
| `/search нейросети для видео` | Поиск по каналам (топ-10 постов) |
| `/digest` | Прислать дайджест сейчас |
| `/digest time 09:00` | Присылать дайджест каждый день в 09:00 |

### Прочее
| Команда | Что делает |
|---|---|
| `/auto_reply` / `/auto_reply off` | Включить/выключить авто-ответ с памятью диалога |
| `/allow @name` (или в ответ на сообщение) | Разрешить другому человеку пользоваться `/ask` |
| `/allow @name 50` | Разрешить, но не больше 50 сообщений за раз |
| `/allow remove` | Забрать доступ |
| `/song текст` | Печать с эффектом набора |
| `/help` | Полный список команд |

## 📡 Настройка каналов (по желанию)

Для функций `/search` и `/digest` нужен список каналов. Есть готовый пример `channels.example.json`.

- Либо автоматически найти ИИ-каналы среди ваших подписок:
  ```bash

  ```

Файл `channels.json` создаётся автоматически и **не** попадает в репозиторий (он личный).

## 🔧 Частые проблемы (FAQ)

**«python: команда не найдена»**
Используйте `python3` вместо `python`. На Windows — переустановите Python с галочкой «Add Python to PATH».

**«pip: команда не найдена» или ошибки при установке**
Убедитесь, что активировали виртуальное окружение (в строке терминала должно быть `(.venv)`). См. Шаг 4.

**Бот пишет «проверь api_id/api_hash в .env»**
В файле `.env` не заполнены или заполнены неверно `api_id` / `api_hash`. Вернитесь к Шагу 6 и 8.

**Меня разлогинило / сессия «слетела»**
Скорее всего, бот был запущен **в двух местах одновременно**. Запускайте только в одном. Удалите файлы `session_name.session*`, выполните `python login.py` заново.

**Команды `/ask` не срабатывают**
Бот реагирует только на сообщения **от вашего аккаунта** (или от тех, кому вы дали `/allow`). Пишите команду сами. Проверьте, что в терминале нет ошибок и бот запущен.

**Картинки / голосовые не распознаются (в ответе `[изображение]` или `[аудио сообщение]`)**
Это значит, что не задан ключ **OpenRouter** — он отвечает за распознавание медиа. Текст чата при этом разбирается нормально, бот не падает. Хотите распознавание — добавьте `OPENROUTER_API_KEY` в `.env` (Шаг 7–8) и перезапустите бота.

**Бот пишет «Модель не настроена» или «недоступна — нет ключа провайдера»**
Вы переключились командой `/model` на модель, для которой нет ключа (например, Kimi/GLM — это OpenCode). Вернитесь на DeepSeek командой `/model` → выберите номер DeepSeek, либо добавьте нужный ключ в `.env`.

**Ошибка «No module named telethon» (или другой модуль)**
Не активировано виртуальное окружение. Выполните строку активации из Шага 11 (`Activate.ps1` на Windows / `source .venv/bin/activate` на macOS/Linux) и запустите снова.

**`winget` не распознан (при установке ffmpeg)**
У вас старая версия Windows без winget. Видео-кружки — необязательная функция: просто пропустите Шаг 5, всё остальное будет работать.

**Не могу найти файл `.env` в папке**
Он скрытый — в Проводнике/Finder его обычно не видно, это нормально. Открывайте и редактируйте его командой из Шага 8.2 (`notepad .env` / `open -e .env` / `nano .env`).

**Видео-кружки не распознаются**
Установите `ffmpeg` (Шаг 5). Распознавание кружков также требует ключа **OpenRouter** (см. выше).

**Ответы ИИ обрезаются или приходит ошибка про лимит**
Возможно, выбрана модель с маленьким контекстом, а сообщений слишком много. Уменьшите число в `/ask N` или выберите другую модель через `/model`.
